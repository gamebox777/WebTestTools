# Webシステム用テストツール 要件定義書

## 1. システムの役割と目的
Next.jsを用いた、Webシステムのテスト状況の管理および確認を行うための内部向けツール。
バックエンドとしてデータベースは利用せず、一時的なテストデータ等をモックとして保持・管理する運用を想定。

## 2. 開発環境ならびにデプロイ
- デプロイ先: Vercel
- フレームワーク: Next.js (App Router推奨)
- 言語: TypeScript 推奨
- スタイリング: Tailwind CSS + shadcn/ui

## 3. 主要機能(予定)
- **スタートメニュー（ランチャー）**: 各種テストツールへのアプローチを統合したトップ画面。
- **テスト画像生成ツール**: 指定した形式(PNG, JPG, PDF, XLSX)、サイズ、枚数でダミーファイルを生成する機能 (既存のPythonスクリプト `image_generator.py` をVercel/Next.js上で動くように移植・統合)。
- **テキストカウンタ**: 入力されたテキストの文字数（空白あり/なし）、行数、バイト数等をリアルタイムでカウントして表示する機能。
- **その他のツール**: テキストジェネレータなどを順次追加可能とする拡張性。
