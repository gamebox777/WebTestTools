# よく使うGit/GitHubコマンド一覧

日々の開発でよく使う順番で、GitおよびGitHubのコマンドをまとめました。

## 1. 開発の始まりと終わり

### 最新の状態を取得する (作業開始時)
リモート（GitHub）の最新情報をローカルに反映します。
```sh
git pull origin develop
```

### 変更をステージングエリアに追加する
変更や新規作成したファイルを、コミット対象として登録します。
```sh
# すべての変更を一括で追加（よく使います）
git add .

# 特定のファイルだけを追加
git add <ファイル名>
```

### 変更をコミット（記録）する
変更内容にメッセージを付けて記録します。何を変更したか分かりやすく書くのがコツです。
```sh
git commit -m "コミットメッセージ（例：〇〇機能の追加）"
```

### リモート（GitHub）に変更をPushする (作業完了時)
```sh
git push origin <ブランチ名>
```

## 2. ブランチの操作（分岐と移動）

### 現在のブランチや未コミットの変更を確認する
一番よく使うコマンドです。迷ったらまずはこれを実行します。
```sh
git status
```

### ブランチの一覧を見る
```sh
git branch          # ローカルのブランチのみ
git branch -a       # リモートも含めたすべてのブランチ
```

### 新しいブランチを作って、そこに移動する
```sh
git checkout -b <新しいのブランチ名>
# （または最近のGitだと以下も可能）
git switch -c <新しいのブランチ名>
```

### 既存の別のブランチに移動する
```sh
git checkout <移動先のブランチ名>
# （または）
git switch <移動先のブランチ名>
```

## 3. マージ（統合）の操作

### 他のブランチの変更を現在のブランチに取り込む
例：`develop`ブランチの変更を`main`ブランチに取り込みたい場合
```sh
# 1. まず取り込まれる側（main）に移動
git checkout main

# 2. 統合したいブランチ（develop）をマージ
git merge develop

# 3. 変更をPush
git push origin main
```

## 4. 困ったときのコマンド

### コミットの履歴を見たい
```sh
git log
git log --oneline   # 一行でコンパクトに表示
```

### 変更をもとに戻したい（コミット前）
```sh
# 特定のファイルの変更を取り消す
git checkout -- <ファイル名>
# または
git restore <ファイル名>
```

### 直前のコミットのメッセージを間違えた！
```sh
git commit --amend
```
