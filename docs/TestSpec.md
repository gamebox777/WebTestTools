# Webシステム用テストツール テスト仕様書

## 1. テスト方針
Next.js アプリケーションとしての基本ルーティング、およびUIコンポーネントの表示確認を主とする。
Vercel環境へデプロイした状態での動作確認も本仕様に含む。

## 2. テストケース（予定）
### 2.1 画面表示テスト
- [ ] トップページ (`/`) が正常に表示されること。
- [ ] トップページにマニュアルへのリンクが表示され、クリック時に別タブで正しく開くこと。
- [ ] テキスト一覧ページが正常にレンダリングされること。
- [ ] テキストカウンタ画面 (`/tools/text-counter`) が正常に表示されること。
- [ ] テキストジェネレータ画面 (`/tools/text-generator`) が正常に表示されること。

### 2.2 UI・アクションテスト
- [ ] テキストカウンタ画面にて、テキストエリアへの入力に応じて文字数・行数・バイト数等がリアルタイムに正しく計算・反映されること。
- [ ] テキストジェネレータ画面にて、指定条件（文字種別・半角/全角/混在・改行の有無・文字数）でのテキスト生成とクリップボードへのコピーが正常に機能すること。
- [ ] 選択した「半角」「全角」「混在」オプションが、生成されたテキストに正しく反映されていること。
- [ ] 各種ボタンがデザイン通りに表示され、ホバーやクリックアクションが機能すること。
- [ ] レスポンシブ対応（スマホ・PC幅での崩れがないこと）。

## 3. 自動・手動テスト手段
- **ローカル確認**: `npm run dev` 起動後、ブラウザで要素崩れ・エラーログがないか確認。
- **デプロイ確認**: Vercelへのpush後、プレビューURLにて同様の動作を確認。
